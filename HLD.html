4 · Security Design
ConcernCurrent ImplementationProduction Upgrade
🔑 Auth Random Bearer token in server Map JWT RS256 signed, short expiry + refresh tokens
🛡 Authorization Role check on DELETE — 403 for non-admin RBAC middleware with permission scopes
🚦 Rate Limiting 100 req/min per IP, in-memory Map Redis sliding window, distributed
🌐 CORS Allow-Origin: * for development Restrict to specific domain only
✅ Validation Required fields, parseFloat on amount Zod / Joi schema validation
🔒 Secrets Hardcoded credentials (demo only) Vault / AWS Secrets Manager + env vars
5 · Scalability — Current vs Production
ComponentCurrent (Demo)Production Upgrade Path
🗄 Data Store In-memory Array (resets on restart) PostgreSQL with indexes on date, type, status
⚡ Cache In-memory Map, 5s TTL Redis distributed cache with configurable TTL
🔐 Auth Token Map (lost on restart) JWT + Redis blacklist for revocation
📄 Pagination Offset-based (Array slice) Cursor-based for deep pages at scale
🔍 Search JS Array filter (linear scan) Elasticsearch for full-text search
🖥 Frontend Single bundle, full re-fetch React Query + code splitting + lazy loading
☁️ Infra Single Node process K8s + load balancer + horizontal scaling
6 · Database Schema (Production)
CREATE TABLE transactions (
  id           UUID           PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      UUID           NOT NULL REFERENCES users(id),
  description  VARCHAR(255)   NOT NULL,
  amount       DECIMAL(15,2)  NOT NULL,
  type         ENUM('CREDIT', 'DEBIT')                  NOT NULL,
  status       ENUM('PENDING', 'COMPLETED', 'FAILED')  DEFAULT 'PENDING',
  category     VARCHAR(50),
  currency     CHAR(3)        DEFAULT 'USD',
  created_at   TIMESTAMP      DEFAULT NOW(),
  updated_at   TIMESTAMP      DEFAULT NOW()
);
idx_tx_user_id → transactions(user_id)
idx_tx_type → transactions(type)
idx_tx_status → transactions(status)
idx_tx_created → transactions(created_at DESC)
